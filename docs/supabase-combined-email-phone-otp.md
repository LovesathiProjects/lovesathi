# Supabase Combined Email + Phone OTP Plan

Goal: one OTP is generated by Supabase during signup, then the same code is delivered to the signup email and the signup phone number.

Important Supabase limitation: native Supabase Auth does not send the same signup OTP to email and SMS automatically. To do this, use a Supabase **Send Email Auth Hook**. That hook receives `email_data.token`, which is the email OTP, and can send that same token to both channels.

## What to Configure in Supabase

1. Keep **Authentication > Providers > Email** enabled.
2. Keep **Confirm email** enabled.
3. Go to **Authentication > Auth Hooks**.
4. Create a **Send Email** hook.
5. Use an HTTPS endpoint or Supabase Edge Function as the hook target.
6. Generate and save the hook secret. Supabase sends signed webhook headers, so the endpoint must verify the signature.

## What the Hook Must Do

1. Read `email_data.token`.
2. Read the email from `user.email`.
3. Read the phone from `user.user_metadata.phone`.
4. Send `email_data.token` to the email address.
5. Send the same `email_data.token` to the phone number through Twilio or another SMS provider.
6. Return HTTP 200 only if both sends succeed.

## Environment Values Needed

For the hook endpoint:

```env
SEND_EMAIL_HOOK_SECRET=
OTP_EMAIL_FROM=
OTP_EMAIL_PROVIDER_KEY=
TWILIO_ACCOUNT_SID=
TWILIO_AUTH_TOKEN=
TWILIO_MESSAGING_SERVICE_SID=
```

If using Zoho SMTP instead of an email API, we need the Zoho SMTP host, port, username, and app password.

## App Behavior

The app already collects phone only on the Create Account screen. Onboarding should not ask for phone again.

After the hook is connected, the verification page can honestly say the same 6-digit code was sent to both email and phone. The user enters that one code and Supabase verifies it with the email OTP flow.
